# quick start

get started with arbor in just a few minutes. this guide will walk you through account creation, initial setup, and your first AI conversation.

## create your account

<Steps>
  <Step title="sign up">
    visit [app.arbor.xyz](https://app.arbor.xyz) and click **sign up**
    
    you can register with:
    - email address
    - google account
    - github account
  </Step>
  
  <Step title="verify email">
    if using email signup, check your inbox for verification code
  </Step>
  
  <Step title="complete profile">
    add your name and optionally upload an avatar
  </Step>
</Steps>

## initial configuration

### choose your AI provider

arbor supports multiple AI providers. select your preferred option:

<Tabs>
  <Tab title="OpenAI">
    best for: GPT-4o, general purpose AI
    
    1. go to **settings** > **models**
    2. click **OpenAI** section
    3. enter your API key from [platform.openai.com](https://platform.openai.com)
    4. select default model (GPT-4o recommended)
  </Tab>
  
  <Tab title="Anthropic">
    best for: Claude 3.5, coding assistance
    
    1. go to **settings** > **models**
    2. click **Anthropic** section
    3. enter your API key from [console.anthropic.com](https://console.anthropic.com)
    4. select Claude 3.5 Sonnet as default
  </Tab>
  
  <Tab title="arbor credits">
    no API key needed - use arbor's models
    
    1. purchase credits in **settings** > **billing**
    2. models automatically available
    3. pay-as-you-go pricing
  </Tab>
</Tabs>

<Info>
you can configure multiple providers and switch between models anytime
</Info>

## your first chat

### starting a conversation

1. click **new chat** in the sidebar or press `Cmd+N`
2. type your message in the prompt bar
3. press `Enter` to send

### understanding responses

AI responses include:
- **streaming text**: see the answer as it's generated
- **code blocks**: syntax-highlighted code you can copy
- **tool usage**: web search, calculations, etc.
- **outputs**: captured content for later use

## creating your first project

projects help organize related work:

<Steps>
  <Step title="create project">
    1. click **new project** in sidebar
    2. name it (e.g., "my app", "research")
    3. add a description
  </Step>
  
  <Step title="add instructions">
    provide context for AI:
    ```
    I'm building a SaaS application with:
    - Next.js and TypeScript
    - PostgreSQL database
    - Stripe payments
    
    Always use modern React patterns and hooks
    ```
  </Step>
  
  <Step title="upload files">
    add reference documents:
    - requirements.md
    - database schema
    - API documentation
  </Step>
  
  <Step title="start project chat">
    click **new chat** within the project to begin with context
  </Step>
</Steps>

## essential features to explore

### 1. file attachments
enhance conversations with context:
- drag & drop files into chat
- supports text, code, PDFs, images
- AI can analyze and reference content

### 2. model switching
change AI models based on task:
- click model selector in chat header
- GPT-4o for general tasks
- Claude 3.5 for coding
- Gemini for fast responses

### 3. sharing conversations
share your work with others:
1. click share button in chat
2. set expiration (1 day, 7 days, 30 days)
3. copy link to share

### 4. keyboard shortcuts
work faster with shortcuts:

| action | shortcut |
|--------|----------|
| new chat | `Cmd/Ctrl + N` |
| search | `Cmd/Ctrl + K` |
| focus prompt | `/` |
| settings | `Cmd/Ctrl + ,` |

### troubleshooting quick fixes

<AccordionGroup>
  <Accordion title="API key not working">
    - verify key is correct (no extra spaces)
    - check API account has credits
    - ensure key has proper permissions
    - try generating a new key
  </Accordion>
  
  <Accordion title="slow responses">
    - switch to faster model (GPT-4o-mini)
    - check internet connection
    - reduce attachment sizes
    - clear browser cache
  </Accordion>
  
  <Accordion title="chat not loading">
    - refresh the page
    - check browser console for errors
    - try incognito/private mode
    - disable browser extensions
  </Accordion>
</AccordionGroup>

# chat interface

the heart of arbor. powerful ai assistance with streaming responses, tool integration, and persistent memory.

## starting a conversation

### creating a new chat
- click **new chat** in the sidebar or press `cmd+n` (mac) / `ctrl+n` (windows)
- type your message in the prompt bar
- press `enter` or click send

### chat contexts
- **standard chat**: general conversation at root level
- **project chat**: conversations within specific project context
- **workspace chat**: development-focused with code assistance

## sending messages

<Tabs>
  <Tab title="text">
    type and press enter. arbor supports:
    - multi-line messages (shift+enter for new line)
    - markdown formatting
    - code blocks with syntax highlighting
    - mathematical expressions
  </Tab>
  <Tab title="attachments">
    attach files for context:
    1. click the `+` button
    2. select **upload file**
    3. choose files (text, images, pdfs, code)
    4. files are processed with rag for semantic search
  </Tab>
  <Tab title="screenshots">
    capture and share:
    1. click the `+` button
    2. select **take screenshot**
    3. select area to capture
    4. add description with your question
  </Tab>
</Tabs>

## ai responses

### response features
- **streaming**: see the answer as it's generated
- **tool calls**: ai searches web, extracts content, and more
- **code blocks**: formatted with syntax highlighting
- **outputs**: automatically captured for later use

### tool capabilities

| tool | purpose | example |
|------|---------|---------|
| web search | find current information | "what's the latest on..." |
| content extraction | get webpage content | "summarize this article: [url]" |
| calculations | perform math operations | "calculate compound interest..." |
| file analysis | process uploaded documents | "analyze this csv data" |

## memory and context

### conversation memory
arbor remembers everything:
- full conversation history
- context maintained after refresh
- previous messages inform responses

### semantic recall
intelligent memory that:
- recalls relevant past conversations
- surfaces related context automatically
- improves accuracy over time

### project context
when chatting in a project:
- custom instructions automatically applied
- uploaded files provide context
- all project chats share knowledge

## attachments and rag

### supported files
- **text**: `.txt`, `.md`, `.csv`, `.json`, `.xml`
- **code**: `.js`, `.ts`, `.py`, `.java`, `.go`, etc.
- **documents**: `.pdf`, `.docx` (text extracted)
- **images**: `.png`, `.jpg`, `.gif` (visual analysis)

### how rag works
1. files are processed and chunked
2. content embedded using fastembed
3. stored in pgvector for semantic search
4. relevant chunks retrieved during conversation
5. ai uses retrieved context for answers

## managing chats

### organization

#### titles
- auto-generated after first exchange
- rename via menu (`...` button)
- appear in sidebar for navigation

#### search and filter
- command menu (`cmd+k`) to search
- filter by project or date
- find by content

#### archiving
- archive old chats to declutter
- archived chats remain searchable
- restore anytime

### sharing

generate public links:
1. click share button in header
2. choose expiration:
   - 1 day
   - 7 days
   - 30 days
   - never
3. copy generated link
4. manage in settings

<Warning>
shared links are public. anyone with the link can view the conversation.
</Warning>

## model selection

### switching models
click model selector in header to choose:
- **openai**: gpt-4o, gpt-4o-mini
- **anthropic**: claude 3.5 sonnet, claude 3.5 haiku
- **google**: gemini 2.0 flash
- **openrouter**: 100+ models

### model strengths
- **gpt-4o**: general purpose, multimodal
- **claude 3.5**: coding, analysis, writing
- **gemini**: fast responses, efficiency

## best practices

### effective prompting
- be specific and clear
- provide context and examples
- break complex tasks into steps
- use attachments for reference

### managing context
- keep related discussions in same chat
- use projects for topic organization
- clear chat when switching topics
- leverage semantic memory

### performance tips
- use appropriate models
- attach files vs pasting content
- enable tools for current info
- organize with projects

## troubleshooting

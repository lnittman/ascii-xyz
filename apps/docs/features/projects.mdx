# projects

projects in arbor help you organize related conversations, files, and instructions for better context management and workflow organization.

## understanding projects

### what are projects?
projects are containers that group:
- related chat conversations
- custom instructions for AI behavior
- reference files and documentation
- shared context across chats

### when to use projects
create projects for:
- **specific applications**: "my SaaS app", "portfolio website"
- **research topics**: "market analysis", "technical research"
- **client work**: separate projects per client
- **learning paths**: "learning rust", "AI course"

## creating projects

### quick create
1. click **new project** in the sidebar
2. enter project name
3. optionally add description
4. click create

### detailed setup
<Steps>
  <Step title="basic information">
    - **name**: clear, descriptive title
    - **description**: project purpose and goals
    - **image**: optional project icon/cover
  </Step>
  <Step title="add instructions">
    provide custom instructions that apply to all project chats:
    ```
    you are helping me build a React Native app.
    use TypeScript and Expo.
    follow these conventions:
    - functional components only
    - use styled-components
    - implement proper error handling
    ```
  </Step>
  <Step title="upload files">
    add reference files for context:
    - documentation
    - specifications
    - code samples
    - design mockups
  </Step>
</Steps>

## project features

### custom instructions

#### writing effective instructions
```markdown
# project context
building an e-commerce platform with Next.js and Stripe

# technical requirements
- use TypeScript with strict mode
- follow REST API conventions
- implement proper error handling
- write tests for critical paths

# code style
- use functional components
- prefer composition over inheritance
- follow eslint-config-airbnb

# specific guidelines
- always validate user input
- implement loading states
- handle edge cases
- consider mobile-first design
```

#### instruction scope
- instructions apply to all chats within the project
- override default AI behavior
- provide consistent context
- ensure aligned responses

### file management

#### supported file types
- **documentation**: .md, .txt, .pdf
- **code**: all programming languages
- **data**: .json, .csv, .xml
- **images**: .png, .jpg, .gif

### project chats

#### creating project chats
1. navigate to project page
2. click **new chat** within project
3. chat inherits project context automatically

#### chat organization
- all project chats appear in project view
- sidebar shows project association
- easy navigation between related chats
- shared memory across project chats

## best practices

### organization tips
- use clear, descriptive names
- group related projects with prefixes
- archive completed projects
- regularly update instructions
- keep file references current

### instruction guidelines
- be specific and detailed
- include technical requirements
- define coding standards
- specify output formats
- mention what to avoid

### file management
- keep files under 10MB each
- use text formats when possible
- organize files logically
- update outdated files
- remove irrelevant content

## troubleshooting

<AccordionGroup>
  <Accordion title="files not being used">
    - ensure files are properly uploaded
    - check file format compatibility
    - verify files contain relevant content
    - try re-uploading
  </Accordion>
  <Accordion title="instructions ignored">
    - make instructions clear and specific
    - avoid contradictory guidelines
    - check instruction length limits
    - test with simpler instructions
  </Accordion>
  <Accordion title="project context missing">
    - ensure you're chatting within project
    - refresh the page
    - check project settings
    - verify files are uploaded
  </Accordion>
</AccordionGroup>
